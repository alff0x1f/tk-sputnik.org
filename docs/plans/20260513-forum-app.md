# Forum App — phpBB Archive Read-Only

## Overview

Создать Django-app `apps/forum` для отображения архива phpBB-форума (2007–2014) в read-only режиме.
Данные импортируются из MySQL phpBB в PostgreSQL одноразовой management-командой; runtime-зависимости от MySQL нет.
Страница `/forum/` показывает категории и подфорумы с их статистикой.

## Context (from discovery)

- `apps/forum_import/` — уже существует, MySQL-подключение `DATABASES['phpbb']` настроено
- `apps/demo/templates/demo/forum.html` — готовый HTML-шаблон с хардкодом; нужно адаптировать
- `config/urls.py` — подключить `path('forum/', include('apps.forum.urls'))`
- `config/settings.py` — добавить `'apps.forum'` в `INSTALLED_APPS`
- phpBB 3.x, таблица `vu2_forums`, префикс `vu2_`
- Категорий 5 (forum_type=0): Приключения, Велосипед, Федерация, Разное, Из рук в руки
- Подфорумы (forum_type=1): прямые дети категорий + вложенные depth=2
- На странице индекса показываем только прямых детей категорий

## Solution Overview

1. `apps/forum` — модели `ForumCategory` и `SubForum` в PostgreSQL
2. `apps/forum_import/management/commands/import_phpbb_forums.py` — читает `vu2_forums` из MySQL, пишет в PostgreSQL через `update_or_create`
3. View `forum_index` — `ForumCategory.objects.prefetch_related('subforums')`, в шаблоне фильтр по `phpbb_parent_id`
4. Шаблон адаптирован из `demo/forum.html`, цвета категорий (`cat-sky/earth/steel/wine`) назначаются через `forloop.counter|divisibleby` или cycle

## Technical Details

**phpBB поля → модели:**

| phpBB | Модель |
|---|---|
| forum_id | phpbb_id (unique) |
| parent_id | phpbb_parent_id (raw) |
| forum_name | name |
| forum_desc | description |
| forum_topics | topic_count |
| forum_posts | post_count |
| forum_last_post_subject | last_post_title |
| forum_last_poster_name | last_post_username |
| forum_last_post_time | last_post_at (Unix ts → datetime) |
| left_id | sort_order |

**FK category для вложенных подфорумов:**
Строим `{forum_id: row}` словарь. Для каждого type=1: если parent — категория (type=0) → прямая связь. Если parent — subforum → рекурсивно ищем корневую категорию.

## Post-Completion

**Запуск импорта (ручной шаг):**
```bash
# MySQL должен быть запущен
docker compose --profile phpbb up -d phpbb_db
# дождаться healthy, затем:
uv run python manage.py import_phpbb_forums
# ожидаемый вывод: Imported 5 categories, N subforums
```

**Ручная проверка:**
- Открыть `/forum/` в браузере
- Убедиться, что отображаются 5 категорий с подфорумами
- Проверить статистику тем/сообщений и дату последнего поста

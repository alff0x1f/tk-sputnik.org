# ТЗ для kolco24.ru: API эндпоинт для sync_contributors

## Описание

Нужен API-эндпоинт, который отдаёт данные о периодах взносов, участниках и платежах.
Эти данные синхронизируются в локальную БД сайта tk-sputnik.org через management command
`sync_contributors`, запускаемый по cron.

## Эндпоинт

- Метод: GET
- URL: `/api/contributors/`
- Content-Type ответа: `application/json`
- Аутентификация: без аутентификации (публичный эндпоинт) или с API-ключом в заголовке
  `Authorization: Bearer <token>` — уточнить с командой

## Формат ответа

```json
{
  "periods": [
    {
      "id": 1,
      "name": "весна 2024",
      "date": "2024-03-01",
      "is_active": true
    }
  ],
  "members": [
    {
      "id": 5,
      "name": "Алексей Костров",
      "label": "Горная школа"
    }
  ],
  "donations": [
    {
      "member_id": 5,
      "period_id": 1,
      "is_paid": true,
      "amount": "1500.00",
      "paid_date": "2024-04-10",
      "recipient": "sbp",
      "note": ""
    }
  ]
}
```

## Описание полей

### periods

| Поле        | Тип     | Описание                                      |
|-------------|---------|-----------------------------------------------|
| `id`        | int     | Первичный ключ периода на kolco24             |
| `name`      | string  | Название периода, например "весна 2024"       |
| `date`      | string  | Дата начала периода в формате YYYY-MM-DD      |
| `is_active` | boolean | True если период активен (принимает взносы)   |

### members

| Поле    | Тип    | Описание                                                              |
|---------|--------|-----------------------------------------------------------------------|
| `id`    | int    | Первичный ключ участника на kolco24                                   |
| `name`  | string | Имя участника                                                         |
| `label` | string | Метка участника (соответствует полю `notes` модели ClubMember на kolco24); свободный текст ("Горная школа", "Вечный участник"); пустая строка если метки нет |

### donations

| Поле        | Тип     | Описание                                                        |
|-------------|---------|-----------------------------------------------------------------|
| `member_id` | int     | ID участника (ссылается на `members[].id`)                      |
| `period_id` | int     | ID периода (ссылается на `periods[].id`)                        |
| `is_paid`   | boolean | True если взнос оплачен                                         |
| `amount`    | string  | Сумма взноса в виде строки с двумя знаками после запятой        |
| `paid_date` | string  | Дата оплаты в формате YYYY-MM-DD; null если не оплачен          |
| `recipient` | string  | Способ оплаты ("sbp", "card", и т.д.); пустая строка если нет  |
| `note`      | string  | Примечание; пустая строка если нет                              |

## Маппинг полей

Поле `label` в ответе API соответствует полю `notes` модели `ClubMember` на kolco24.
На стороне tk-sputnik.org оно сохраняется в поле `label` модели `ClubMember`.
Значение отображается под именем участника на странице `/contributors/` как простой текст.

## Примечания

- Все записи отдаются целиком (без пагинации и фильтрации)
- Порядок записей не важен — сортировка выполняется на стороне клиента
- Пустой массив (`[]`) допустим для любого из трёх ключей
- Поле `paid_date` для неоплаченных взносов должно быть `null`, а не пустой строкой
